## 常见问题解答

### 安装和配置问题

#### Q: 应用程序无法启动，提示缺少依赖项
A: 请确保已安装所有必要的依赖项，执行 `npm install` 命令安装所有依赖。

#### Q: Google Gemini API 密钥如何获取和配置
A: 您需要在 Google Cloud Console 中创建项目并启用 Gemini API，然后生成 API 密钥。将 API 密钥添加到 `.env` 文件中，格式为 `VITE_GEMINI_API_KEY=您的API密钥`。

#### Q: 应用程序需要管理员权限吗
A: 是的，创建 Junction 点需要管理员权限。应用程序会自动检测权限并提示用户以管理员身份运行。

### 迁移操作问题

#### Q: 迁移过程中出现错误，如何处理
A: 请查看终端中的错误信息，根据错误提示进行修复。常见错误包括权限不足、目标路径不存在或被占用。

#### Q: 迁移后应用程序无法正常运行
A: 请检查源路径和目标路径的权限是否正确，确保 Junction 链接已正确创建。如果问题仍然存在，可以尝试使用应用程序的"修复链接"功能。

#### Q: 大型文件迁移速度很慢，如何优化
A: 应用程序已实现分块复制和并行处理来优化大型文件迁移速度。您也可以尝试关闭其他正在运行的应用程序，以释放系统资源。

### AI 分析问题

#### Q: AI 分析失败，提示 API 错误
A: 请检查您的网络连接和 API 密钥是否正确。如果 API 密钥正确但仍然失败，可能是因为超出了 API 配额限制，请稍后再试。

#### Q: AI 分析结果不准确，如何处理
A: AI 分析结果仅供参考，最终决策应由用户做出。如果您认为分析结果不准确，可以尝试重新运行分析，或提供更详细的应用程序信息。

### 权限和安全问题

#### Q: 应用程序需要哪些权限
A: 应用程序需要以下权限：
- 读取和写入文件系统的权限
- 创建和管理符号链接的权限（需要管理员权限）
- 访问网络的权限（用于 AI 分析）

#### Q: 迁移操作是否安全
A: 迁移操作相对安全，因为它会创建 Junction 链接保持原有路径的访问性。但为了安全起见，建议在迁移前备份重要数据。

#### Q: 应用程序会收集用户数据吗
A: 应用程序不会收集用户数据。所有操作都在本地执行，AI 分析的请求会发送到 Google Gemini API，但不会包含个人身份信息。

### 技术问题

#### Q: 应用程序支持哪些 Windows 版本
A: 应用程序支持 Windows 10 及以上版本。

#### Q: 如何查看详细的操作日志
A: 应用程序会在 `logs` 目录中生成详细的操作日志，您可以查看这些日志了解操作的详细过程。

#### Q: 应用程序崩溃或无响应，如何处理
A: 请尝试以下步骤：
1. 关闭应用程序并重新启动
2. 检查系统资源使用情况，确保有足够的内存和磁盘空间
3. 查看日志文件了解崩溃原因
4. 如果问题仍然存在，请通过 GitHub Issues 提交问题报告
